StockGravity UI 디자인 리뷰 요청

# 시스템 개요
한국 주식 자동 분석 시스템. Streamlit 웹 대시보드.

# 워크플로우
매일 15:20 → 2,790개 필터링 → 500개 Stock Pool 저장 → Top 20 AI 분석 → 사용자가 AI Reports에서 Approve → Trading Approval Queue 이동 → 3~7일 모니터링 → 매매 시작 → Active Trades → 거래 완료

# 현재 페이지 구조

## 1. Dashboard (app.py) - 거의 비어있음
- 전체 현황 한눈에 보기 필요

## 2. Stock Pool (500개 종목)
- 사이드바: Status/점수/거래대금/날짜 필터
- 메인: 테이블(30개 페이지네이션) + 선택 시 메모/메트릭/액션 버튼
- 문제: 페이지네이션 불편, 배지 의미 불명확

## 3. AI Reports (Top 20 AI 분석)
- 필터: 추천등급(🟢STRONG_APPROVE/🟡WATCH_MORE/🔴DO_NOT_APPROVE), 리포트상태(🟢ACTIVE/💰TRADED/🔴DROPPED)
- 메인: Expander 카드 20개 (제목 클릭해야 내용 보임)
  - 내용: 요약/메트릭/모멘텀/유동성/리스크 탭/Approve버튼
- 문제: Expander 방식이라 비교 어려움, 20개면 스크롤 너무 김

## 4. Trading (3개 탭)
### Tab1: Approval Queue (approved 종목)
- 메트릭: Queue수/재평가대상/평균점수
- 테이블 선택 → 상세(차트 3개: 가격 캔들+MA, 거래량, RSI) + 액션버튼(매매시작/재평가/제거)
- 문제: 너무 기능 많음

### Tab2: Active Trades (trading 종목) - 현재 비어있음
- 테이블: 진입가/현재가/수익률

### Tab3: Trade History (completed 종목) - 현재 비어있음
- 테이블: 진입가/청산가/수익률

## 5. Stock Detail - 구현 안 됨
- 필요: 종합 차트, AI 리포트 히스토리, 메모, 재평가 로그
